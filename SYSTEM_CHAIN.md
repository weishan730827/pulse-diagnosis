## 二、处理层：双轨辨证引擎

### 2.1 入口文件

| 文件 | 作用 |
|------|------|
| `dual_track_pipeline.py` | **一键入口**。接收 chen_input.json 或 CLI 参数，跑完轨道A+轨道B+交叉验证 |
| `bianzheng_shizhi_v3.py` | **核心引擎**。内含 `differential_diagnosis`（八家）和 `chen_jianguo_analyze`（陈建国） |
| `chen_jianguo_formula_signatures.json` | **50方脉位签名库**。132条结构化脉位签名，供陈建国轨道匹配 |

### 2.2 运行命令

```bash
# 方式1：用 chen_input.json（完整六步填充，推荐）
python dual_track_pipeline.py --input ../temp/chen_input.json

# 方式2：直接用脉象+症状字符串（快速模式，不含陈建国六步匹配）
python dual_track_pipeline.py \
  --pulses "左寸中取无力,左关弦,左尺沉细,右寸弱,右尺沉微弦" \
  --symptoms "胸闷气短,畏寒,后背冷,泡沫尿" \
  --zhiyibang-opts "AA AB AC ..." \
  --tongue-opts "AB BA BE ..."
```

### 2.4 轨道B：陈建国独立辨证

**六步定向法**（陈建国《仲景阴阳脉法》2020）：

| 步骤 | 内容 | 输出 |
|------|------|------|
| 第一步 | 搭双手 → 总体太过/不及 | `step1_overall` |
| 第二步 | 左右对比 → 大法定向 | `step2_direction`（升法/降法 + 四象） |
| 第三步 | 三部逐位 → 太过/不及/正常 | `step3_positions`（6个脉位） |
| 第四步 | 浮中沉分层 → 最强脉动层次 | `step4_layers`（6个脉位） |
| 第五步 | 脉质判定 → 拘急/有力/无力/管细/模糊 | `step5_qualities`（6个脉位） |
| 第六步 | 交叉对比 → 左右强弱/双关/双尺 | `step6_cross` |

**50方脉位签名匹配**（`chen_jianguo_formula_signatures.json`）：

- 50方 × 132条结构化脉位签名
- 按 hand(L/R) + pos(cun/guan/chi) + depth(fu/zhong/chen) + excess_deficiency(excess/deficiency) + weight 逐条比对
- 命中 trend + layer 逐项加分，降序排列输出Top15

### 2.5 交叉验证机制

| 类型 | 定义 | 示例 |
|------|------|------|
| **会合点** ★★★ | 两轨独立指向同一方剂/方向 | 乌梅丸（八家胡希恕厥阴6项 × 陈建国850分Top1） |
| **分叉点** | 八家支持但陈建国因脉位条件不满足而否定 | 小柴胡汤（八家胡希恕少阳8项 × 陈建国需总体太过不满足） |
| **单向点** | 仅一条轨道给出，另一条无对应 | 升陷汤（张锡纯独有，陈建国体系无对应脉位签名但方向一致） |

## 五、从空白到报告的完整操作流程

### Step 1：采集原始数据

- 脉诊：六部浮中沉 + 脉质（拘急/有力/无力/管细/模糊）
- 症状：主诉 + 寒热 + 汗出 + 口渴 + 口苦 + 二便 + 饮食 + 头面 + 四肢 + 面色 + 舌象
- 西医：血糖/影像/化验（如有）

### Step 2：填充三个表格

| 表 | 文件 | 输出 |
|---|------|------|
| 勾选表A（知医邦10维28脉） | `勾选表A_知医邦10维28脉入八家辨证.md` | zhiyibang_opts 列表 |
| 舌诊勾选表 | `舌诊勾选表_知医邦量化表.md` | tongue_opts 列表 |
| 勾选表B（陈建国六步定向） | `勾选表B_陈建国六步定向50方脉证经方.md` | chen_input.json |

### Step 3：构建 chen_input.json

- 将勾选表B的六步结果按 §1.3 的 JSON schema 填入
- 将症状、舌象、西医结果填入 `symptoms` 字段
- 保存为 `chen_input.json`

### Step 4：运行双轨流水线

```bash
cd output/
python dual_track_pipeline.py --input ../temp/chen_input.json
```

### Step 5：阅读产物 + 临证决策

- 先读 `cross_validate_*.txt` 看会合点（最高确定性方向）
- 再读分叉点，理解两轨不一致的原因
- 参照 `医案_*.md` 中的三线整合方案
- 处方后记录反馈，走完闭环
